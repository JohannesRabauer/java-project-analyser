= Introduction and Goals

== Requirements Overview

The Java Project Analyser ingests a Java codebase, drives a ten-phase LLM analysis pipeline, and persists the results as a RAG (Retrieval-Augmented Generation) knowledge base in PostgreSQL.

The system has two operating modes:

**Analyse mode** — Phases 1–9 extract and synthesise knowledge from the source code.
Phase 10 chunks every phase result, embeds each chunk, and stores it in the vector store.
Inputs arrive through a Docker volume mount (read-only) or by cloning a Git repository at analysis time.
A Vaadin browser UI drives the analysis and displays the static analysis report.

**Query mode** — Once a job is `INDEXED`, the Vaadin UI exposes four interactive query capabilities that use retrieval-augmented generation against the stored knowledge base:

* Risk Analysis (security, usability, readability)
* Documentation Generation (audience-tailored)
* LLM-Assisted Bug Finding
* LLM-Assisted Feature Implementation Guidance

== Quality Goals

Top quality goals, in priority order.
Each maps to a named architectural approach in <<Solution Strategy>>.

[cols="1,2,3,2", options="header"]
|===
| ID | Quality Characteristic | Scenario (short) | Architectural Approach

| QG-1
| Modifiability
| A developer switches the LLM provider from OpenAI to local Ollama by changing one configuration property — no code changes.
| Hexagonal architecture with `LlmPort` and `EmbeddingPort` (ADR-004); LangChain4J multi-provider abstraction (ADR-003)

| QG-2
| Deployability
| A developer who clones the repository can run the full application with `docker compose --profile local up` in under 5 minutes.
| Docker Compose with named profiles (ADR-005); self-contained Quarkus container image; pgvector bundled in the PostgreSQL image (ADR-008)

| QG-3
| Functional Correctness
| For a well-known project (e.g., Spring PetClinic), the report correctly identifies all framework technologies and the architectural style; risk analysis correctly surfaces at least one real security concern.
| Phase-by-phase LLM prompting; uncertainty markers preserved (BR-009); structured output schema validation; RAG retrieval grounding (ADR-008)

| QG-4
| Time Behaviour
| A 500-class project completes all 10 phases (including RAG indexing) and has all query capabilities available within 12 minutes.
| Chunked batching (BR-005); parallel chunk dispatch; phase result caching (BR-002); batch embedding in Phase 10

| QG-5
| Query Relevance
| A bug-finding query for a real defect in the analysed project returns the responsible class in the top-3 results in ≥ 80 % of test cases.
| pgvector cosine similarity with configurable threshold (BR-019); BR-022 architectural context injection for feature queries

| QG-6
| Usability
| A non-technical project manager can run a risk analysis and export the report in under 3 minutes without reading any documentation.
| Vaadin query views with guided input; structured, audience-aware output; one-click export (UC-004, UC-005)
|===
