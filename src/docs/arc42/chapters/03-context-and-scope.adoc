= Context and Scope

== Business Context

The Java Project Analyser is a headless, multi-transport Model Context Protocol (MCP) server. It sits between external AI systems (such as Cursor, VS Code, or Claude Desktop) and the LLMs/database that execute analysis and search capabilities.
It abstracts away prompt engineering, parsing, caching, vector indexing, and report rendering behind a standard protocol interface.

The system supports two operating modes:

. **Analyse** — a ten-phase pipeline extracts knowledge from a Java codebase (from filesystem or Git) and indexes it into the PostgreSQL RAG database.
. **Query** — the client AI asks questions or executes semantic retrieval over codebase chunks via the standard MCP tool and resource endpoints.

A separate **Vaadin MCP Client Demo** connects to the server as an external client, demonstrating how an LLM utilizes the MCP server tools to interact with a human.

== External Systems

[cols="1,2,3", options="header"]
|===
| System | Direction | Responsibility

| PostgreSQL + pgvector
| Internal (owned by this deployment)
| Primary database.
Stores analysis job metadata, LLM call logs, phase results, RAG chunks with vector embeddings, and query results.
The pgvector extension enables cosine similarity search over stored embeddings (ADR-008).

| Ollama
| Outbound (local profile)
| Runs open-weight LLMs (Llama 3, Mistral, nomic-embed-text, etc.) on the host machine.
Receives phase prompts and embedding requests; returns structured text and float vectors.

| OpenAI API
| Outbound (openai profile)
| Hosted GPT-4o / text-embedding-3-small or equivalent.
Receives phase prompts and embedding requests; returns structured text and float vectors.

| Anthropic API
| Outbound (anthropic profile)
| Hosted Claude 3.5 Sonnet or equivalent.
Receives phase prompts; returns structured text.
NOTE: Anthropic does not yet provide a general embedding endpoint; for this profile the system uses Ollama (locally) or a configurable secondary embedding provider for Phase 10.

| Git Repository
| Outbound (optional)
| Remote source repository.
Cloned to a temporary directory when the user provides a URL instead of a pre-mounted volume.

| Source Project (Docker Volume)
| Inbound (read-only)
| The Java codebase being analysed.
Mounted at `/workspace` inside the application container.
|===

== Technical Context

[cols="2,1,3", options="header"]
|===
| Channel | Protocol | Notes

| Browser ↔ Application
| HTTP on port 8080
| Vaadin Flow uses WebSocket for server-push progress updates during both the analysis pipeline and RAG indexing.

| Application → PostgreSQL
| JDBC (port 5432)
| All persistent state: jobs, phase results, RAG chunks, query results. pgvector `<=>` operator for similarity queries.

| Application → Ollama (LLM)
| HTTP on port 11434
| REST API of the Ollama server for chat and embedding calls.

| Application → OpenAI
| HTTPS on port 443
| OpenAI Chat Completions API and Embeddings API.

| Application → Anthropic
| HTTPS on port 443
| Anthropic Messages API.

| Application → Git
| Git over HTTPS or SSH
| Delegated to `git clone`; credentials from the host environment.

| Application → Workspace volume
| POSIX filesystem
| Read-only bind mount; no writes to the analysed project (BR-006).
|===
