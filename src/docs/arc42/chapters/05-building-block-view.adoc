== Building Block View

=== Level 1: Docker Compose Stack

The system deploys as a Docker Compose stack with three containers and two named volumes.

[plantuml,level1-containers,svg]
----
@startuml
!include <C4/C4_Container>

LAYOUT_WITH_LEGEND()

title Level 1 – Docker Compose Stack

Person(agent, "AI Agent", "Claude Code, Cursor, VS Code Copilot")

System_Boundary(stack, "java-project-analyser (Docker Compose)") {
    Container(app, "App", "Quarkus 3.x / Java 21", "MCP server: analyses Java projects, serves pre-computed facts and on-demand LLM reasoning")
    ContainerDb(pg, "PostgreSQL + pgvector", "PostgreSQL 16", "Stores knowledge graph, RAG embeddings, analysis results")
    Container(ollama, "Ollama", "Ollama", "Local LLM inference for pipeline, query, and embedding models")
    Container(workspace_vol, "workspace volume", "Docker Volume", "Mounted project source code")
    Container(cache_vol, "cache volume", "Docker Volume", "Persists analysis cache between runs")
}

Rel(agent, app, "MCP JSON-RPC 2.0", "stdio / SSE")
Rel(app, pg, "SQL + pgvector", "JDBC")
Rel(app, ollama, "HTTP", "Ollama API")
Rel(app, workspace_vol, "reads", "filesystem")
Rel(app, cache_vol, "reads/writes", "filesystem")

@enduml
----

[cols="1,3,2,2"]
|===
| Building Block | Responsibility | Interface | Source Location

| App
| Runs the MCP server, orchestrates analysis pipeline, serves queries
| MCP JSON-RPC 2.0 (stdio + SSE), HTTP landing page
| `src/main/java/`

| PostgreSQL + pgvector
| Persists knowledge graph nodes/edges, RAG vector embeddings, analysis metadata
| JDBC / SQL with pgvector extensions
| `src/main/resources/db/migration/`

| Ollama
| Provides local LLM inference (pipeline model, query model, embedding model)
| HTTP REST (Ollama API)
| External container (configuration in `docker-compose.yml`)

| workspace volume
| Makes the target project's source tree available read-only to the app container
| POSIX filesystem mount
| `docker-compose.yml` volume definition

| cache volume
| Persists intermediate analysis results across container restarts
| POSIX filesystem mount
| `docker-compose.yml` volume definition
|===

=== Level 2: App Internals

The app container follows a hexagonal (ports & adapters) architecture. Inbound adapters accept requests, application services orchestrate workflows, domain services implement analysis logic, and outbound adapters integrate with infrastructure through ports.

[plantuml,level2-app-internals,svg]
----
@startuml
!include <C4/C4_Component>

LAYOUT_WITH_LEGEND()

title Level 2 – App Internals (Hexagonal Architecture)

Container_Boundary(app, "App Container") {

    ' --- Inbound Adapters ---
    Component(mcp_stdio, "MCP Stdio Handler", "Quarkus", "Reads/writes MCP JSON-RPC over stdin/stdout")
    Component(mcp_sse, "MCP SSE Controller", "Quarkus REST", "Exposes MCP JSON-RPC over HTTP Server-Sent Events")
    Component(landing, "Thymeleaf Landing Page", "Thymeleaf", "Renders status page and MCP integration instructions")

    ' --- Application Services ---
    Component(pipeline_svc, "AnalysisPipelineService", "Java", "Orchestrates the full analysis pipeline: parse, analyse, summarise, index")
    Component(mcp_dispatch, "McpDispatcher", "Java", "Routes MCP tool calls to the appropriate domain service or pre-computed result")

    ' --- Domain Services ---
    Component(ast_svc, "JavaParserAstService", "JavaParser", "Parses Java source into AST, extracts structural facts")
    Component(graph_svc, "GraphAnalysisService", "Java", "Builds and queries the class/package dependency graph")
    Component(static_svc, "StaticAnalysisService", "Java", "Runs static checks: Checkstyle warnings, code smells")
    Component(llm_summary_svc, "LlmSummaryService", "Java", "Generates natural-language summaries via LLM for classes, modules, project")
    Component(rag_idx_svc, "RagIndexingService", "Java", "Chunks code and facts, computes embeddings, stores in vector DB")

    ' --- Outbound Ports ---
    Component(llm_port, "LlmPort", "Java Interface", "Port for LLM completion requests")
    Component(embedding_port, "EmbeddingPort", "Java Interface", "Port for embedding vector computation")
    Component(rag_port, "RagPort", "Java Interface", "Port for vector storage and similarity search")
    Component(project_port, "ProjectSourcePort", "Java Interface", "Port for reading project source files")
    Component(cache_port, "CachePort", "Java Interface", "Port for reading/writing analysis cache")

    ' --- Outbound Adapters ---
    Component(lc4j_llm, "LangChain4J LLM Adapter", "LangChain4J", "Implements LlmPort via LangChain4J (Ollama/OpenAI)")
    Component(lc4j_embed, "LangChain4J Embedding Adapter", "LangChain4J", "Implements EmbeddingPort via LangChain4J embedding models")
    Component(pg_rag, "PostgreSQL RAG Adapter", "jOOQ + pgvector", "Implements RagPort: stores/retrieves vectors in PostgreSQL")
    Component(fs_project, "Filesystem Project Adapter", "Java NIO", "Implements ProjectSourcePort: reads from workspace volume")
    Component(git_project, "Git Project Adapter", "JGit", "Implements ProjectSourcePort: reads from Git repositories")
    Component(fs_cache, "Filesystem Cache Adapter", "Java NIO", "Implements CachePort: reads/writes cache volume")
}

==== Inbound Adapters

[cols="1,3,2,2"]
|===
| Building Block | Responsibility | Interface | Source Location

| MCP Stdio Handler
| Reads MCP JSON-RPC requests from stdin, writes responses to stdout; used when the agent launches the server as a subprocess
| stdin/stdout (JSON-RPC 2.0)
| `src/main/java/…​/adapter/inbound/stdio/`

| MCP SSE Controller
| Exposes the MCP protocol over HTTP Server-Sent Events at `/mcp/sse`; used by agents that connect via network
| HTTP GET `/mcp/sse` (SSE stream)
| `src/main/java/…​/adapter/inbound/sse/`

| Thymeleaf Landing Page
| Renders a human-readable status page at `/` showing server health and MCP integration instructions
| HTTP GET `/` (HTML)
| `src/main/java/…​/adapter/inbound/web/`
|===

==== Application Services

[cols="1,3,2,2"]
|===
| Building Block | Responsibility | Interface | Source Location

| AnalysisPipelineService
| Orchestrates the end-to-end analysis pipeline: triggers AST parsing, graph construction, static analysis, LLM summarisation, and RAG indexing in sequence
| `analyse(projectPath): AnalysisResult`
| `src/main/java/…​/application/AnalysisPipelineService.java`

| McpDispatcher
| Maps incoming MCP tool names to domain service calls; returns pre-computed facts for instant tools, delegates to LLM for on-demand tools
| `dispatch(toolName, params): McpResponse`
| `src/main/java/…​/application/McpDispatcher.java`
|===

==== Domain Services

[cols="1,3,2,2"]
|===
| Building Block | Responsibility | Interface | Source Location

| JavaParserAstService
| Parses Java source files into AST nodes, extracts class metadata, method signatures, field declarations, and relationships
| `parse(sourceFile): CompilationUnit`
| `src/main/java/…​/domain/service/JavaParserAstService.java`

| GraphAnalysisService
| Builds an in-memory dependency graph from parsed AST facts; answers queries about class relationships, module boundaries, and architecture patterns
| `buildGraph(facts): DependencyGraph`
| `src/main/java/…​/domain/service/GraphAnalysisService.java`

| StaticAnalysisService
| Runs static analysis checks (Checkstyle rules, code smell detection) and produces structured findings
| `analyse(compilationUnits): List<Finding>`
| `src/main/java/…​/domain/service/StaticAnalysisService.java`

| LlmSummaryService
| Sends structured prompts to the LLM to generate natural-language summaries for classes, modules, and the overall project
| `summarise(context): Summary`
| `src/main/java/…​/domain/service/LlmSummaryService.java`

| RagIndexingService
| Chunks code and analysis facts, computes embedding vectors, and stores them for later semantic search retrieval
| `index(facts): void`
| `src/main/java/…​/domain/service/RagIndexingService.java`
|===

==== Outbound Ports

[cols="1,3,2,2"]
|===
| Building Block | Responsibility | Interface | Source Location

| LlmPort
| Abstracts LLM completion requests so domain services are decoupled from any specific LLM provider
| `complete(prompt): String`
| `src/main/java/…​/domain/port/outbound/LlmPort.java`

| EmbeddingPort
| Abstracts embedding vector computation so the indexing service is provider-agnostic
| `embed(text): float[]`
| `src/main/java/…​/domain/port/outbound/EmbeddingPort.java`

| RagPort
| Abstracts vector storage and similarity search operations
| `store(embedding, metadata): void`, `search(query, k): List<Result>`
| `src/main/java/…​/domain/port/outbound/RagPort.java`

| ProjectSourcePort
| Abstracts reading project source files regardless of origin (filesystem or Git)
| `listFiles(path): List<Path>`, `readFile(path): String`
| `src/main/java/…​/domain/port/outbound/ProjectSourcePort.java`

| CachePort
| Abstracts reading and writing intermediate analysis results to persistent cache
| `get(key): Optional<byte[]>`, `put(key, value): void`
| `src/main/java/…​/domain/port/outbound/CachePort.java`
|===

==== Outbound Adapters

[cols="1,3,2,2"]
|===
| Building Block | Responsibility | Interface | Source Location

| LangChain4J LLM Adapter
| Implements `LlmPort` using LangChain4J's ChatLanguageModel abstraction, supporting Ollama and OpenAI backends
| Implements `LlmPort`
| `src/main/java/…​/adapter/outbound/llm/`

| LangChain4J Embedding Adapter
| Implements `EmbeddingPort` using LangChain4J's EmbeddingModel abstraction
| Implements `EmbeddingPort`
| `src/main/java/…​/adapter/outbound/embedding/`

| PostgreSQL RAG Adapter
| Implements `RagPort` using jOOQ for SQL and pgvector for similarity search against PostgreSQL
| Implements `RagPort`
| `src/main/java/…​/adapter/outbound/rag/`

| Filesystem Project Adapter
| Implements `ProjectSourcePort` by reading files from the mounted workspace volume via Java NIO
| Implements `ProjectSourcePort`
| `src/main/java/…​/adapter/outbound/project/`

| Git Project Adapter
| Implements `ProjectSourcePort` by cloning or reading from a Git repository via JGit
| Implements `ProjectSourcePort`
| `src/main/java/…​/adapter/outbound/project/`

| Filesystem Cache Adapter
| Implements `CachePort` by reading/writing serialised analysis results to the mounted cache volume
| Implements `CachePort`
| `src/main/java/…​/adapter/outbound/cache/`
|===
