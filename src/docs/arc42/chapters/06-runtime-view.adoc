= Runtime View

Every building block from <<Building Block View>> appears in at least one scenario below.
Scenarios cover the happy path and one error/recovery case (SC-06).

---

== SC-01: Analyse Project via MCP

An external AI agent calls the `analyse_project` tool over MCP stdio transport.
The pipeline orchestrates phases 1–8, caching each result, and returns the `jobId` immediately.

Demonstrates: MCP Stdio Transport, Tool Dispatcher, Analysis Pipeline Service, Filesystem Project Adapter, Phase Executors 1–8, LangChain4J LLM Adapter, Filesystem Cache Adapter, RAG Indexing Service, LangChain4J Embedding Adapter, PostgreSQL RAG Adapter.

[plantuml, sc-01-analyse-mcp, svg]
....
@startuml
skinparam monochrome true
skinparam defaultFontSize 11
autonumber

participant "AI Agent\n(Claude Code / Cursor)" as Agent
participant "MCP Stdio Transport" as Transport
participant "Tool Dispatcher" as Dispatcher
participant "Analysis Pipeline Service" as Pipeline
participant "Filesystem Project Adapter" as FSProject
participant "Phase Executors [1..8]" as Phase
participant "Prompt Template Loader" as Prompts
participant "LangChain4J LLM Adapter" as LlmAdapter
participant "ChatLanguageModel\n(Ollama|OpenAI|Anthropic)" as LLM
participant "Filesystem Cache Adapter" as Cache
participant "RAG Indexing Service" as RagIndexer
participant "LangChain4J Embedding Adapter" as EmbedAdapter
participant "PostgreSQL RAG Adapter" as RagAdapter
database "PostgreSQL\n(pgvector)" as DB

Agent -> Transport : JSON-RPC 2.0\n{method: "tools/call",\nparams: {name: "analyse_project",\narguments: {path: "/workspace"}}}
Transport -> Dispatcher : dispatch("analyse_project", args)
Dispatcher -> Pipeline : start(AnalysisConfig)
Pipeline --> Dispatcher : jobId (async)
Dispatcher --> Transport : {result: {jobId: "abc-123",\nstatus: "RUNNING"}}
Transport --> Agent : JSON-RPC 2.0 response

Pipeline -> FSProject : loadProject(LocalSource{"/workspace"})
FSProject --> Pipeline : ProjectTree

loop phase = 1 to 8
  Pipeline -> Phase : execute(job, projectTree)
  Phase -> Prompts : loadTemplate(phaseId)
  Prompts --> Phase : PromptTemplate
  Phase -> LlmAdapter : executePrompt(PhasePrompt)
  LlmAdapter -> LLM : chat(messages)
  LLM --> LlmAdapter : response text
  LlmAdapter -> LlmAdapter : parse + validate JSON schema
  LlmAdapter --> Phase : PhaseResponse
  Phase --> Pipeline : PhaseResult
  Pipeline -> Cache : save(PhaseResult)
end

Pipeline -> RagIndexer : index(AnalysisJob)
RagIndexer -> EmbedAdapter : embedBatch(chunks)
EmbedAdapter --> RagIndexer : List<float[]>
RagIndexer -> RagAdapter : index(jobId, chunksWithVectors)
RagAdapter -> DB : INSERT INTO rag_chunks
DB --> RagAdapter : OK
RagIndexer --> Pipeline : IndexSummary

Pipeline -> DB : UPDATE analysis_jobs\nSET status = 'INDEXED'
@enduml
....

== SC-02: Pre-computed Tool Query (Instant)

An agent calls `get_class_purpose` for a previously analysed class.
The dispatcher retrieves the cached result directly — no LLM call required.

Demonstrates: MCP Stdio Transport, Tool Dispatcher, Query Orchestration Service, PostgreSQL RAG Adapter.

[plantuml, sc-02-precomputed, svg]
....
@startuml
skinparam monochrome true
skinparam defaultFontSize 11
autonumber

participant "AI Agent" as Agent
participant "MCP Stdio Transport" as Transport
participant "Tool Dispatcher" as Dispatcher
participant "Query Orchestration Service" as QuerySvc
participant "PostgreSQL RAG Adapter" as RagAdapter
database "PostgreSQL\n(pgvector)" as DB

Agent -> Transport : JSON-RPC 2.0\n{method: "tools/call",\nparams: {name: "get_class_purpose",\narguments: {className: "AnalysisPipelineService"}}}
Transport -> Dispatcher : dispatch("get_class_purpose", args)
Dispatcher -> QuerySvc : getClassPurpose(jobId, className)
QuerySvc -> RagAdapter : searchByText(jobId,\n"AnalysisPipelineService purpose", topK=1)
RagAdapter -> DB : SELECT chunk_text, metadata\nFROM rag_chunks\nWHERE job_id = $1\nAND metadata->>'className' = $2\nLIMIT 1
DB --> RagAdapter : RagResult (cached phase output)
RagAdapter --> QuerySvc : List<RagResult>
QuerySvc --> Dispatcher : ClassPurpose DTO
Dispatcher --> Transport : {result: {className: "...",\npurpose: "...", layer: "..."}}
Transport --> Agent : JSON-RPC 2.0 response
@enduml

== SC-03: On-demand Tool Query (LLM-powered)

An agent calls `get_possible_bugs`. The dispatcher retrieves relevant RAG chunks,
submits them with a bug-analysis prompt to the query LLM, and returns structured findings.
This takes 10–30 seconds depending on model and context size.

Demonstrates: MCP Stdio Transport, Tool Dispatcher, Query Orchestration Service, Bug Finder Service, LangChain4J Embedding Adapter, PostgreSQL RAG Adapter, LangChain4J LLM Adapter, Prompt Template Loader.

[plantuml, sc-03-ondemand, svg]
....
@startuml
skinparam monochrome true
skinparam defaultFontSize 11
autonumber

participant "AI Agent" as Agent
participant "MCP Stdio Transport" as Transport
participant "Tool Dispatcher" as Dispatcher
participant "Query Orchestration Service" as QuerySvc
participant "Bug Finder Service" as BugSvc
participant "Prompt Template Loader" as Prompts
participant "LangChain4J Embedding Adapter" as EmbedAdapter
participant "EmbeddingModel" as EmbedModel
participant "PostgreSQL RAG Adapter" as RagAdapter
database "PostgreSQL\n(pgvector)" as DB
participant "LangChain4J LLM Adapter" as LlmAdapter
participant "ChatLanguageModel\n(Query Model)" as LLM

Agent -> Transport : JSON-RPC 2.0\n{method: "tools/call",\nparams: {name: "get_possible_bugs",\narguments: {scope: "all"}}}
Transport -> Dispatcher : dispatch("get_possible_bugs", args)
Dispatcher -> QuerySvc : findBugs(jobId, scope)
QuerySvc -> BugSvc : findBugs(jobId, scope)

BugSvc -> EmbedAdapter : embed("common bug patterns\nnull safety concurrency")
EmbedAdapter -> EmbedModel : POST /embed
EmbedModel --> EmbedAdapter : float[]
EmbedAdapter --> BugSvc : queryVector

BugSvc -> RagAdapter : search(jobId, queryVector, topK=15)
RagAdapter -> DB : SELECT chunk_text, metadata\nFROM rag_chunks\nORDER BY embedding <=> $vector\nLIMIT 15
DB --> RagAdapter : List<RagResult>
RagAdapter --> BugSvc : List<RagResult>

BugSvc -> Prompts : loadTemplate("bug-analysis")
Prompts --> BugSvc : PromptTemplate
BugSvc -> LlmAdapter : executePrompt(bugPrompt + chunks)
LlmAdapter -> LLM : chat(messages)
LLM --> LlmAdapter : structured JSON\n(suspect locations, severity, rationale)
LlmAdapter -> LlmAdapter : parse + validate JSON schema
LlmAdapter --> BugSvc : PhaseResponse

BugSvc --> QuerySvc : BugFindings
QuerySvc --> Dispatcher : BugFindings DTO
Dispatcher --> Transport : {result: {bugs: [...]}}
Transport --> Agent : JSON-RPC 2.0 response
@enduml
....

== SC-04: Resume After Interruption (Cache Reuse)

The analysis pipeline was interrupted (process kill, timeout, or crash) after completing phases 1–5.
On restart, the pipeline detects cached results and resumes from phase 6 without re-executing earlier phases or repeating LLM calls.

Demonstrates: Analysis Pipeline Service, Filesystem Cache Adapter, Phase Executors 6–8, LangChain4J LLM Adapter, RAG Indexing Service.

[plantuml, sc-04-resume, svg]
....
@startuml
skinparam monochrome true
skinparam defaultFontSize 11
autonumber

participant "AI Agent" as Agent
participant "MCP Stdio Transport" as Transport
participant "Tool Dispatcher" as Dispatcher
participant "Analysis Pipeline Service" as Pipeline
participant "Filesystem Cache Adapter" as Cache
participant "Phase Executors [6..8]" as Phase
participant "LangChain4J LLM Adapter" as LlmAdapter
participant "ChatLanguageModel" as LLM
participant "RAG Indexing Service" as RagIndexer

Agent -> Transport : JSON-RPC 2.0\n{method: "tools/call",\nparams: {name: "analyse_project",\narguments: {path: "/workspace"}}}
Transport -> Dispatcher : dispatch("analyse_project", args)
Dispatcher -> Pipeline : start(AnalysisConfig)

Pipeline -> Cache : listCompleted(jobId)
Cache --> Pipeline : [phase-1, phase-2, phase-3,\nphase-4, phase-5]

note right of Pipeline
  Phases 1–5 restored from cache.
  No LLM calls repeated.
end note

loop phase = 1 to 5
  Pipeline -> Cache : load(jobId, phaseN)
  Cache --> Pipeline : PhaseResult (from disk)
end

loop phase = 6 to 8
  Pipeline -> Phase : execute(job, projectTree)
  Phase -> LlmAdapter : executePrompt(PhasePrompt)
  LlmAdapter -> LLM : chat(messages)
  LLM --> LlmAdapter : response text
  LlmAdapter --> Phase : PhaseResponse
  Phase --> Pipeline : PhaseResult
  Pipeline -> Cache : save(PhaseResult)
end

Pipeline -> RagIndexer : index(AnalysisJob)
RagIndexer --> Pipeline : IndexSummary

Pipeline --> Dispatcher : jobId, status: INDEXED
Dispatcher --> Transport : {result: {jobId: "abc-123",\nstatus: "INDEXED"}}
Transport --> Agent : JSON-RPC 2.0 response
@enduml

== SC-05: Git Clone Source

When the agent provides a Git URL instead of a local path, the Git Project Adapter clones the repository to a temporary directory and delegates to the Filesystem Project Adapter.

Demonstrates: Analysis Pipeline Service, Git Project Adapter, Filesystem Project Adapter, Application Configuration.

[plantuml, sc-05-git-clone, svg]
....
@startuml
skinparam monochrome true
skinparam defaultFontSize 11
autonumber

participant "Analysis Pipeline Service" as Pipeline
participant "Application Configuration" as Config
participant "Git Project Adapter" as GitAdapter
participant "git binary" as Git
participant "Filesystem Project Adapter" as FSAdapter

Pipeline -> Config : resolveSourceType(source)
Config --> Pipeline : GIT_URL

Pipeline -> GitAdapter : loadProject(GitSource{url, branch})
GitAdapter -> Git : git clone --depth 1\n--branch <branch>\n<url> /tmp/workspace-<uuid>
Git --> GitAdapter : exit 0 (clone complete)

GitAdapter -> FSAdapter : loadProject(\nLocalSource{"/tmp/workspace-<uuid>"})
FSAdapter -> FSAdapter : traverse src/main/java,\nsrc/test/java, build files
FSAdapter --> GitAdapter : ProjectTree
GitAdapter --> Pipeline : ProjectTree
@enduml

== SC-06: Cold Start — Analysis Not Ready (Error/Recovery)

An agent calls a query tool before any analysis has completed for the given project.
The dispatcher returns a structured MCP error response indicating the prerequisite is not met.
This is the system's primary error/recovery scenario.

Demonstrates: MCP Stdio Transport, Tool Dispatcher, Query Orchestration Service, PostgreSQL RAG Adapter.

[plantuml, sc-06-cold-start, svg]
....
@startuml
skinparam monochrome true
skinparam defaultFontSize 11
autonumber

participant "AI Agent" as Agent
participant "MCP Stdio Transport" as Transport
participant "Tool Dispatcher" as Dispatcher
participant "Query Orchestration Service" as QuerySvc
participant "PostgreSQL RAG Adapter" as RagAdapter
database "PostgreSQL\n(pgvector)" as DB

Agent -> Transport : JSON-RPC 2.0\n{method: "tools/call",\nparams: {name: "get_class_purpose",\narguments: {className: "Foo"}}}
Transport -> Dispatcher : dispatch("get_class_purpose", args)
Dispatcher -> QuerySvc : getClassPurpose(jobId, "Foo")
QuerySvc -> RagAdapter : searchByText(jobId, "Foo purpose", topK=1)
RagAdapter -> DB : SELECT ... FROM rag_chunks\nWHERE job_id = $1
DB --> RagAdapter : empty result set

RagAdapter --> QuerySvc : [] (no chunks found)
QuerySvc --> QuerySvc : detect: no indexed analysis\nfor this project

QuerySvc --> Dispatcher : AnalysisNotReadyException

Dispatcher --> Transport : {error: {\ncode: -32002,\nmessage: "Analysis not ready",\ndata: {\nreason: "NO_INDEXED_ANALYSIS",\nhint: "Call analyse_project first"\n}}}
Transport --> Agent : JSON-RPC 2.0 error response

note right of Agent
  Agent receives structured error.
  Expected recovery: call analyse_project,
  then retry the query.
end note
@enduml
....
