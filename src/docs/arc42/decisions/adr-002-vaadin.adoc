== Context

The original ADR-002 chose Vaadin Flow as a browser UI for driving analysis runs and presenting reports.
The product vision has changed: the system is now a headless MCP server.
AI agents (Claude Code, Cursor) are the primary interface, not a browser.

A minimal landing page is still needed to show server status, analyzed projects, and integration instructions.

== Decision

Thymeleaf server-rendered landing page at `localhost:8080/`.

"No UI at all" scored higher but loses the ability to show copy-pasteable MCP config snippets to new users — the primary onboarding path.

== Consequences

* Single HTML template + minimal CSS. No WebSocket, no server push, no session state.
* Page shows: health status, list of analyzed projects, and MCP config snippets for Claude Code / Cursor.
* Vaadin dependency removed entirely from `pom.xml`.
* No Vaadin TestBench or Playwright testing needed — a simple HTTP integration test suffices.
