== Overview

This directory holds the complete specification for the Java Project Analyser.
The specification covers all user goals, system interfaces, business rules, acceptance criteria, and supplementary models.

For the product requirements and problem statement, see link:prd.adoc[prd.adoc].

The system operates in two sequential modes:

* **Analyse mode** (UC-001, UC-002, UC-003) — ten-phase LLM pipeline that extracts knowledge from a Java project and indexes it into the RAG knowledge base.
* **Query mode** (UC-004 through UC-007) — interactive query capabilities backed by semantic retrieval from the indexed knowledge base.

== Actors

[cols="1,3", options="header"]
|===
| Actor | Description

| Software Architect
| Reviews an acquired or inherited system before committing to an architectural change; runs risk analysis.

| Onboarding Developer
| Gets up to speed on a service they have just joined; uses documentation generation and bug-finding queries.

| Technical Writer
| Produces documentation for a system they did not build; uses the generated documentation as a first draft.

| Project Manager
| Understands technical scope before planning a migration or audit; reviews risk analysis results.

| Developer (working on analysed project)
| Describes a bug or new feature in natural language and receives codebase-grounded guidance.

| LLM API (System Actor)
| External language model that processes phase prompts and query prompts; returns structured analysis and query results.

| Embedding Model (System Actor)
| External or local embedding model (Ollama / OpenAI) that converts text to float vectors for RAG indexing and similarity search.
|===
