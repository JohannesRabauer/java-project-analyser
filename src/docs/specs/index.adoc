== Overview

This directory holds the complete specification for the Java Project Analyser.
The specification covers all user goals, system interfaces, business rules, acceptance criteria, and supplementary models.

For the product requirements and problem statement, see link:prd.adoc[prd.adoc].

The system operates as an **MCP Server**, supporting two primary interactions:

* **Analyse mode** (UC-001, UC-002, UC-003) — ten-phase LLM pipeline that extracts knowledge from a Java project and indexes it into the RAG knowledge base.
* **Query mode** (UC-004 through UC-007) — interactive query capabilities backed by semantic retrieval from the indexed knowledge base, exposed as MCP tools.

== Actors

[cols="1,3", options="header"]
|===
| Actor | Description

| External AI Client
| Primary Client Actor. An AI assistant, autonomous agent, or IDE extension (e.g., Cursor, Claude Desktop) that initiates MCP connections, calls tools, and reads resources.

| Developer
| Secondary Actor. Interacts with the AI client (e.g., in a chat panel) or logs into the Vaadin Demo Client, which uses the MCP server.

| LLM API (System Actor)
| External language model that processes phase prompts and query prompts; returns structured analysis and query results.

| Embedding Model (System Actor)
| External or local embedding model (Ollama / OpenAI) that converts text to float vectors for RAG indexing and similarity search.
|===
