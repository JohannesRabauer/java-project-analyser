== Command Signature

[source,shell]
----
java-analyser analyse <project> [options]
----

=== Positional Arguments

[cols="1,1,4", options="header"]
|===
| Argument | Required | Description

| `<project>`
| Yes
| Local filesystem path to the project root, or a git repository URL (`https://` or `ssh://`).
|===

=== Options

[cols="2,1,1,3", options="header"]
|===
| Option | Type | Default | Description

| `--output-dir <path>`
| path
| `./analysis-output`
| Directory where the report and JSON artifacts are written.
Created if it does not exist.

| `--cache-dir <path>`
| path
| `./analysis-cache`
| Directory where phase result files and the LLM call log are persisted.
Created if it does not exist.

| `--format <adoc\|md>`
| enum
| `adoc`
| Report output format.

| `--model <model-id>`
| string
| From config
| LLM model identifier to use.
Overrides the configured default.

| `--phases <range>`
| string
| `all`
| Comma-separated list of phase numbers or ranges to execute (e.g., `1-5`, `1,3,5`).
Use `all` to execute all nine phases.

| `--batch-size <n>`
| integer
| `50`
| Maximum number of classes per LLM batch during Phase 4.
Overrides <<../business-rules.adoc#BR-005,BR-005>> default.

| `--no-build-tool`
| flag
| false
| Skip build-tool dependency resolution; parse build descriptor directly.
Forces the <<../business-rules.adoc#BR-010,BR-010>> fallback.
|===

== Input Validation

[cols="1,3,2", options="header"]
|===
| Input | Validation Rule | Error on Failure

| `<project>` (local path)
| Path must exist and the process must have read permission.
Must contain a `pom.xml` or `build.gradle` at the root or in a direct subdirectory.
| Exit code 2; error to stderr.

| `<project>` (URL)
| Must be a syntactically valid git URL.
`git` must be available on PATH (<<../business-rules.adoc#BR-015,BR-015>>).
| Exit code 1 (missing git) or 2 (bad URL); error to stderr.

| `--output-dir`, `--cache-dir`
| If the path exists it must be a directory with write permission.
If it does not exist the parent must be writable.
| Exit code 1; error to stderr.

| `--format`
| Must be one of: `adoc`, `md`.
| Exit code 1; error to stderr listing accepted values.

| `--model`
| Must be a recognized model identifier.
| Exit code 1; error to stderr listing accepted values.

| `--phases`
| Each value must be an integer in 1–9.
Ranges must be ascending.
| Exit code 1; error to stderr.

| `--batch-size`
| Must be a positive integer ≥ 1.
| Exit code 1; error to stderr.
|===

== Processing

The system executes the full 9-phase analysis pipeline as described in UC-001.
When `--phases` restricts the run to a subset, only the specified phases execute; the report is generated from the available phase results and marked as partial.

Progress updates are written to stdout as each phase starts and completes.

All LLM calls follow the retry and context-window rules in <<../business-rules.adoc#BR-004,BR-004>> and <<../business-rules.adoc#BR-016,BR-016>>.

== Output

=== Files Written

[cols="2,3", options="header"]
|===
| File | Description

| `<output-dir>/report.adoc` (or `.md`)
| Full analysis report.

| `<output-dir>/dependency-graph.json`
| Directed dependency graph (nodes: modules/packages/classes; edges: dependency relationships).

| `<output-dir>/responsibility-map.json`
| Mapping of business and technical responsibilities to owning components.

| `<cache-dir>/phase-N-result.json`
| Structured result for phase N (one file per phase, N = 1–9).

| `<cache-dir>/llm-calls.jsonl`
| Append-only log of all LLM calls (<<../business-rules.adoc#BR-007,BR-007>>).
|===

=== Stdout

Progress lines during execution:

[source]
----
[Phase 1/9] Application Purpose Analysis... done (3.2s)
[Phase 2/9] Module Analysis... done (5.1s)
...
Analysis complete. Report: ./analysis-output/report.adoc
----

=== Exit Codes

[cols="1,3", options="header"]
|===
| Code | Meaning

| 0 | All requested phases completed; report generated.
| 1 | Invalid input or configuration.
| 2 | Project source not found or inaccessible.
| 3 | LLM API error after all retries.
| 4 | Unexpected internal error.
| 130 | Terminated by SIGTERM or SIGINT; partial cache flushed.
|===

== Error Responses

All error messages are written to stderr in the format:

[source]
----
ERROR [code]: <human-readable description>
----

Example:

[source]
----
ERROR [2]: Project not found at path '/projects/nonexistent'.
Ensure the path exists and is readable.
----
