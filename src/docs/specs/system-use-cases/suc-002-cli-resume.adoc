== Command Signature

[source,shell]
----
java-analyser resume <cache-dir> [options]
----

=== Positional Arguments

[cols="1,1,4", options="header"]
|===
| Argument | Required | Description

| `<cache-dir>`
| Yes
| Path to the cache directory produced by a previous `analyse` run.
|===

=== Options

All options from <<suc-001-cli-analyse.adoc#,SUC-001>> are accepted and apply to the remaining phases.
The following option is specific to `resume`:

[cols="2,1,1,3", options="header"]
|===
| Option | Type | Default | Description

| `--force`
| flag
| false
| Bypass the project source mismatch confirmation prompt (<<../use-cases/uc-002-resume-analysis.adoc#,UC-002 extension 2b>>).
|===

== Input Validation

[cols="1,3,2", options="header"]
|===
| Input | Validation Rule | Error on Failure

| `<cache-dir>`
| Directory must exist and be readable.
Must contain at least one file matching `phase-*-result.json` with valid JSON content.
| Exit code 1; error to stderr.

| Phase result files
| Each present file must be valid JSON matching the phase result schema.
Invalid files are treated as absent; the corresponding phase will be re-executed.
| Warning to stderr; execution continues.

| Project source (from cache metadata)
| The `projectSource.resolvedLocalPath` stored in the cache must still be accessible, unless `--force` is set.
| Exit code 2 or user confirmation required.
|===

== Processing

. The system reads all valid `phase-N-result.json` files from `<cache-dir>`.
. The system identifies the set of completed phases and reports them to stdout.
. The system skips completed phases, loading their results directly from cache.
. The system executes remaining phases in order, following the same LLM call, retry, and caching rules as SUC-001.

== Output

Same file outputs as <<suc-001-cli-analyse.adoc#,SUC-001>>.

=== Stdout (example)

[source]
----
Resuming analysis from ./analysis-cache
Completed phases (loaded from cache): 1, 2, 3
Remaining phases to execute: 4, 5, 6, 7, 8, 9
[Phase 4/9] Class Analysis... done (12.4s)
...
Analysis complete. Report: ./analysis-output/report.adoc
----

=== Exit Codes

Same as <<suc-001-cli-analyse.adoc#,SUC-001>>.

== Error Responses

Same format as SUC-001.
Additional error specific to `resume`:

[source]
----
ERROR [1]: No recoverable phase results found in './bad-cache'.
Run 'java-analyser analyse' to start a new analysis.
----
